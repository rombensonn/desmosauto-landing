# ДесмосАвто Landing

Одностраничный лендинг для `desmosauto.ru`: сайты и лендинги для автосервисов, шиномонтажей, детейлинг-центров и смежного автобизнеса.

Стек:

- HTML5
- CSS3
- Vanilla JavaScript
- PHP 8+
- Без базы данных
- Telegram Bot API для отправки заявок

## Структура

```text
desmosauto-landing/
  assets/
    css/styles.css
    js/main.js
    img/placeholders/
  api/lead.php
  config/config.example.php
  storage/.gitkeep
  index.php
  README.md
```

## Локальный запуск

```bash
cd desmosauto-landing
php -S localhost:8000
```

Откройте `http://localhost:8000`.

## Настройка Telegram-бота

1. Откройте Telegram и найдите `@BotFather`.
2. Отправьте команду `/newbot`.
3. Задайте имя и username бота.
4. Скопируйте выданный токен.
5. Напишите сообщение созданному боту от того аккаунта или группы, куда должны приходить заявки.
6. Получите `chat_id`. Самый простой способ для личного чата:

```text
https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates
```

Для группы добавьте бота в группу, отправьте сообщение в группу и снова вызовите `getUpdates`.

## Конфигурация

Скопируйте пример:

```bash
cp config/config.example.php config/config.php
```

Вставьте значения:

```php
const TELEGRAM_BOT_TOKEN = 'ваш_token_бота';
const TELEGRAM_CHAT_ID = 'ваш_chat_id';
```

Файл `config/config.php` нельзя публиковать в репозиторий. Он уже добавлен в `.gitignore`.

## Проверка формы

1. Запустите локальный сервер.
2. Настройте `config/config.php`.
3. Откройте страницу.
4. Заполните форму и отправьте заявку.
5. Проверьте, что в Telegram пришло сообщение с именем, телефоном, типом бизнеса, ситуацией, выбранными улучшениями, комментарием и UTM-данными.

Форма отправляется через `fetch` без перезагрузки страницы. Backend возвращает JSON:

```json
{
  "success": true,
  "message": "Заявка отправлена. Мы свяжемся и уточним детали."
}
```

## Защита и обработка

`api/lead.php`:

- принимает только POST;
- валидирует обязательные поля;
- проверяет телефон;
- проверяет согласие на обработку данных;
- использует honeypot-поле `website_url`;
- отклоняет слишком быструю отправку через `form_started_at`;
- очищает данные перед отправкой;
- не показывает пользователю технические детали Telegram-ошибок;
- хранит токен и `chat_id` только на сервере.

## Что нельзя публиковать

- `config/config.php`
- реальные токены Telegram
- локальные файлы в `storage/`
- персональные данные из тестовых заявок

## Что заменить перед релизом

- контакты в footer;
- финальную политику обработки персональных данных;
- домен и OG-данные, если отличаются от `desmosauto.ru`;
- реальные фото работ и материалов автосервисов;
- реальные кейсы только после получения разрешения;
- финальные тексты про сроки, формат демо и коммерческие условия.
