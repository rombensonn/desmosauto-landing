# 05. Дизайн-система

## Общий визуальный характер

Дизайн ДесмосАвто - это светлый премиальный editorial/SaaS стиль, адаптированный под автомобильный сервисный бизнес. Он должен выглядеть современно, дорого, спокойно и технически надежно, но без ощущения "корпоративной бездушности".

Ключевые ощущения:

- точность;
- чистота;
- скорость;
- доверие;
- технологичность;
- практичность;
- визуальная глубина через 3D-модели и анимации.

## Палитра

Основные цвета:

- почти черный: `#080808`, `#101010`;
- белый: `#ffffff`;
- теплые светлые поверхности: `#fbfbfa`, `#f7f7f5`, `#f6f7f8`;
- линии: `#e4e4df`, `#e2e2df`;
- главный акцент: `#ff5a1f`;
- сильный оранжевый: `#e44810`;
- мягкий оранжевый: `#fff0e8`;
- destructive: `#dc2626`;
- success: `#0f8f5f`;
- projects accent: `#ef5a24`.

Правила:

- основной фон светлый, теплый, нейтральный;
- оранжевый использовать для CTA, акцентов, важных маркеров;
- черный использовать для главного действия, текста и премиальных темных блоков;
- не вводить случайные фиолетовые/синие AI-градиенты;
- не превращать сайт в однотонную бежевую тему.

## Типографика

Главная часть сайта:

- Arial / Helvetica Neue / system-ui;
- тяжелые заголовки;
- компактный line-height;
- Georgia italic как `serif-accent` внутри больших заголовков.

Страница проектов:

- Manrope для основного текста и заголовков;
- Geist Mono для чисел и статистики;
- более editorial/catalog ощущение.

Правила:

- крупные заголовки использовать только в hero/section-head;
- в карточках держать заголовки плотными, но не гигантскими;
- не использовать отрицательный letter-spacing вне уже существующего projects-стиля;
- сохранять читаемость на мобильных.

## Сетка и размеры

Базовые паттерны:

- `.container-page` - основной контейнер;
- `.section-y` - вертикальный ритм секций;
- 8px radius для основных карточек лендинга;
- 14px radius для карточек проектов;
- clamp-based размеры для адаптивности;
- стабильные размеры у карточек, медиа, кнопок и форм.

Не делать:

- карточки внутри декоративных карточек;
- случайные floating sections;
- декоративные blobs/orbs;
- layout-shift при hover;
- слишком большие hero-заголовки внутри компактных компонентов.

## Компоненты

Основные классы:

- `.surface` - светлая карточка с бордером и мягкой тенью;
- `.surface-dark` - темная карточка;
- `.btn-primary` - черная pill-кнопка с оранжевым hover;
- `.btn-secondary`, `.btn-muted`, `.btn-hero-secondary` - дополнительные действия;
- `.eyebrow` - маленькая pill-метка;
- `.tag` - метка/атрибут;
- `.card-hover` - мягкий hover-lift.

## Темные секции

Темные секции нужны для ритма и премиального контраста. Они особенно подходят:

- процессу работы;
- AEO-блокам;
- преимуществам;
- смысловым разделителям.

Темная секция не должна становиться тяжелой. Использовать белый текст с прозрачностью для вторичного текста и оранжевые/теплые акценты.

## Доступность

Нужно сохранять:

- видимый focus-visible;
- достаточный контраст;
- touch-target около 44-48px;
- нормальные labels у форм;
- reduced-motion поведение.
