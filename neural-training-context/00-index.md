# 00. Карта учебной базы

Эта база описывает проект ДесмосАвто как целостную систему: продукт, коммерческий смысл, аудиторию, дизайн, анимации, технологический стек, backend и правила дальнейшей работы.

## Главная идея

ДесмосАвто - это сайт и продуктовая упаковка для продажи сайтов автосервисам и смежным автомобильным бизнесам. Проект не должен восприниматься как абстрактный digital-агентский лендинг. Он должен ощущаться как практический инструмент для владельца автосервиса: сайт объясняет услуги, снимает тревогу клиента, показывает доверие и приводит короткие заявки.

## Основные документы

- `01-project-essence.md` - смысл проекта, позиционирование, ключевая ценность.
- `02-audience-offer-funnel.md` - целевая аудитория, боли, оффер, путь к заявке.
- `03-content-architecture.md` - архитектура страниц и блоков.
- `04-copywriting-semantics.md` - правила текста, SEO/AEO, тональность.
- `05-design-system.md` - цвета, типографика, сетки, компоненты.
- `06-visual-assets-3d.md` - ассеты, 3D PNG, превью кейсов, правила использования.
- `07-animation-system.md` - GSAP, Framer Motion, hover, sticky stack, WebGL.
- `08-ux-forms-conversion.md` - UX-правила, форма заявки, ошибки, доступность.
- `09-technical-stack.md` - Next.js, React, TypeScript, PHP, статический экспорт.
- `10-backend-telegram-deployment.md` - обработчик заявок, Telegram, архив для REG.RU.
- `11-quality-maintenance-rules.md` - как поддерживать проект без расползания стиля.
- `12-short-project-description.md` - короткий бриф, который можно дать модели перед работой.

## Как использовать для обучения модели

Модель должна усвоить три слоя:

1. Смысловой слой: для кого сайт, что он обещает и какие проблемы решает.
2. Визуальный слой: как сайт должен выглядеть, двигаться и ощущаться.
3. Технический слой: как он устроен, собирается, деплоится и принимает заявки.

При дальнейшей работе модель должна не просто редактировать файлы, а сохранять единство этих слоев.
