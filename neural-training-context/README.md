# Учебный контекст проекта ДесмосАвто

Эта папка создана как база знаний для обучения другой нейросети контексту проекта. Файлы не являются кодом сайта и не должны автоматически применяться к интерфейсу. Их задача - объяснить смысл, структуру, визуальный язык, технологический стек и рабочие правила проекта.

## Как читать

Рекомендуемый порядок:

1. `00-index.md` - карта всей базы знаний.
2. `01-project-essence.md` - суть проекта в одном документе.
3. `02-audience-offer-funnel.md` - аудитория, оффер, воронка.
4. `03-content-architecture.md` - страницы, блоки, структура смыслов.
5. `04-copywriting-semantics.md` - тон, формулировки, SEO/AEO-смысл.
6. `05-design-system.md` - дизайн-система.
7. `06-visual-assets-3d.md` - визуальные ассеты и 3D-модели.
8. `07-animation-system.md` - все типы анимаций.
9. `08-ux-forms-conversion.md` - UX, формы и конверсия.
10. `09-technical-stack.md` - технический стек.
11. `10-backend-telegram-deployment.md` - backend, Telegram, деплой.
12. `11-quality-maintenance-rules.md` - правила качества и поддержки.
13. `12-short-project-description.md` - краткий текстовый бриф проекта.

## Важное про секреты

В этой папке не хранится полный Telegram bot token. Реальные секреты должны оставаться в локальном ignored-файле `backend/api/leads.config.php` и попадать только в готовый deployment-архив, когда пользователь просит полностью готовую сборку.
